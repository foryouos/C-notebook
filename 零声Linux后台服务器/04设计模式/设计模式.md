# 设计模式

> 设计模式实在软件开发中，经过`验证的`，用于`解决特定环境`下，`重复出现`的，`特定问题的解决`方案
>
> `解决问题`的`固定套路`，慎用设计模式
>
> * 满足设计原则后，`慢慢迭代`出来
> * 前提: 具体的需求既有`稳定点`，又有`变化点`



# 生成器模式

>化繁为简，逐个击破。分步骤创建复杂的对象，并允许使用相同的代码生成不同类型和形式的对象，称为生成器模式(也叫建造者模式)
